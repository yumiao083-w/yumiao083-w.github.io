## 安全隐患排查

### 🔴 高危

1. **`parse_config()` 使用 `exec()` 执行 config.py**（L852）
   - 如果攻击者能修改 config.py 内容（通过 save API），可以执行任意 Python 代码
   - `submit_config` 和 `save_section` 都能写 config.py

2. **`start_bot` / `stop_bot` 无认证**（L433/L464）
   - 任何人可以远程启动/停止微信机器人
   - `start_bot` 执行 `subprocess.Popen(['python', 'bot.py'])`

3. **voice_call.py 全部路由无认证**
   - `/voice` `/api/voice/*` 所有接口都不需要登录
   - 攻击者可以直接调用 API 消耗你的 LLM/TTS/Whisper 额度
   - 可以读取/删除通话记录

4. **登录密码明文存储在 config.py**
   - 密码 `yumiao0803` 对任何能读 config.py 的人可见

### 🟡 中危

5. **`ast.literal_eval` 处理用户输入**（L658/L876/L1143）
   - 虽然比 `eval` 安全，但处理不当可能导致拒绝服务

6. **多个 API 路由无认证**
   - `/bot_status` — 暴露机器人运行状态
   - `/memory_calendar` — 暴露用户记忆数据
   - `/api/get_memory_detail` — 暴露具体记忆内容
   - `/api/log` — 接收日志

7. **`secret_key = os.urandom(24).hex()`** 每次重启随机生成
   - 重启后所有 session 失效（不是安全问题，但影响体验）

8. **ngrok 暴露整个服务**
   - 任何人知道 URL 就能访问登录页面
   - 爆破密码无速率限制

### 🟢 建议修复

- [ ] voice_call.py 所有路由加 `@login_required`
- [ ] `start_bot` / `stop_bot` 加 `@login_required`
- [ ] 记忆相关 API 加 `@login_required`
- [ ] 登录加暴力破解保护（5次错误锁定）
- [ ] 公开版做独立路由，和私人功能完全隔离
- [ ] ngrok 加 basic auth 或限制只暴露公开路由

### 需求
两种模式：
1. **开袋即食模式** — 我配好所有 API + 角色，用户进来直接通话
   - 不保存任何用户数据（一次性通话）
   - 可选填昵称/话题（不存服务器）
   - 提示词只保留人设预设（去掉私设）
   - 内置 API Key，前端完全看不到

2. **自定义模式** — 用户自己配 LLM/TTS/Whisper API 和提示词
   - 配置存浏览器 localStorage（不传服务器）
   - 我们只提供框架

### 安全考虑
- ngrok 目前暴露整个网页服务器（含配置页面），有安全风险
- 公开版需要独立路由 `/voice/public`，不经过 `@login_required`
- 私人配置和公开版完全隔离
- 需要限流防刷（每 IP 每日 N 次）
