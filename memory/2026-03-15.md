**Phase 3：Bug 修复与调优（测试中发现）**

1. **获取模型报错 `Unexpected token '<'`**
   - 原因：fetch 没检查 response 状态，后端错误返回 HTML 被当 JSON 解析
   - 修复：所有 fetch 加 `r.ok` 检查 + 后端错误处理优化

2. **精筛报错 `'"id"'`**
   - 原因：`.format()` 遇到记忆索引里的花括号 `{` `}` 炸了（KeyError）
   - 修复：改用 `.replace()` 逐个替换变量，不被内容花括号干扰

3. **精筛只返回 1 条**
   - 原因：① 只发了 `user` 消息，模型把检索指令当聊天在回复 ② max_tokens 太低
   - 修复：
     - API 调用加 `system` 角色消息（"你是检索系统不是聊天助手"）
     - 提示词重写：强调不要回复用户、发散联想策略、不限条数
     - max_tokens 500 → 4000（前端可配）
     - 去掉 `[:top_k]` 截断，返回 AI 输出的全部结果

**最终效果**：
- LLM 精筛成功返回 14 条相关记忆（徒步测试场景）
- 总 token 约 66.8k，比之前多 ~5k，可接受
- 强相关排前面，弱相关排后面，检索质量远超关键词匹配
